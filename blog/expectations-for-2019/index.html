對 2019 的期許

2019年1月1日 · 閱讀時間約 3 分鐘
HK Chen
Cloud Engineer

去年 2018 年對於自己的學習評價算是滿意，花了約一個月的時間熟練了 Vue，以及大大小小 SE 的一些事情和工具的安裝及使用，雖然本身並不是這個職位，但我認為這並不該是局限於 SE 該做的事情。

而對於自己 project 的 CI/CD 流程也提昇了不少，因為今年在一場社群演講聽到了也有在掃引入 3rd-party library 的一些免費或收費的軟體來加入 CI/CD 裡面，原來當你個別使用這些 library 的時候他們在法律上是可以來當做商業用途的，但當一起的時候就會有衝突的可能，算是收穫之一，那這些東西就是來避免有法律糾紛的問題也是很重要的。

除了期許今年可以做更多的 side project 出來，也列一些我在今年度希望可以學習或是達成的目標！

學習清單​

前端技術​

  • 使用並學習 React 到目前 Vue 的熟悉階段
  • 比較 Vue 與 React 的開發體驗差異
  • 建立 React 相關的 side project

後端技術​

  • Node.js 和 Go 的學習以及 Python 的精進
  • Java: RabbitMQ 訊息處理, 以及利用 MyBatis 試著搭建 gRPC 的框架
  • 今年在年底完成了利用 MyBatis 搭配 MySQL 建出了 GraphQL 的框架，並且列出什麼樣的情境和框架下搭配前者還是後者會比較好的列表和心得

監控系統​

熟悉並比較出 Zabbix 以及 LibreNMS 的實際區別，目前已經實裝 Zabbix 並搭配 Grafana 進階圖型介面，也安裝 zabbix-java-gateway 來監控 Tomcat 或 WildFly 的一些效能，例如 SQL 的效能監控時間。

而 LibreNMS 有一個 plugin 是可以模擬 billing 系統，可以根據開的大小規格來計算大約價錢，目前是會以模擬 AWS S3 的方式來實作。

而 JMX 也就是 Zabbix 在監控 Java 應用程式的部份，會在近期分享記錄的安裝方式，我覺得蠻多地方要小心的，而且比較冷門的東西，往往文章都是複製來複製去，希望在這邊做一個區隔並且有用的分享文。

Side Projects​

寫出更多的 side project 來解決我在 coding 上遇到的一些麻煩。

今年的期望​

除了增進自己的能力之外，希望可以培養出看商業模式的眉角以及考取 KCA 和 AWS 的一些證照並且...戰勝惰性。

證照目標​

  • KCA (Kubernetes Certified Administrator)
  • AWS 相關認證

個人成長​

  • 培養商業模式分析能力
  • 戰勝惰性 💪
